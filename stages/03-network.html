阶段 3 · 网络互联

前面我们讲了单台服务器里的 GPU 和内存。但大模型训练不是一台服务器能搞定的——需要成百上千台服务器里的几千张 GPU 同时工作。这些 GPU 之间怎么"对话"？数据怎么在它们之间高速流转？这就是网络层要解决的问题。

场景：几千张 GPU 怎么协同工作？

假设你要训练一个千亿参数的大模型。你买了 1000 台 GPU 服务器，每台 8 张 GPU——总共 8000 张 GPU。

训练过程是这样的：每一轮"学习"（专业术语叫一次迭代），所有 8000 张 GPU 各自处理一小批数据，算出各自的"答案"（梯度）。然后，每张 GPU 需要把自己的计算结果分享给其他所有 GPU，并收集其他 GPU 的结果，综合起来更新模型参数。

这种"所有人把自己的数据发给所有人，同时接收所有人的数据"的通信模式，叫做 All-Reduce（全归约）。它不是一次性的——每一轮迭代都会发生，而训练一个大模型可能需要几百万次迭代。

关键数字：在大规模训练集群中，GPU 有 30%–50% 的时间在等网络数据传输，而不是在计算。换句话说，你花大价钱买来的 GPU，有三分之一到一半的时间在"发呆"。网络带宽直接决定了这些 GPU 实际能发挥多少性能。

这不是小问题。如果 8000 张 GPU 的利用率从 90% 掉到 60%，等效于你白买了 2400 张 GPU。所以网络不是配角——它是训练效率的第一要素之一。

核心概念

RDMA：绕开 CPU 的"直达通道"

要理解 AI 集群网络，首先要理解 RDMA（Remote Direct Memory Access，远程直接内存访问）。

普通网络传输的路径是这样的：

  1. GPU 算完数据，存到自己的显存里
  2. CPU 把数据从显存拷到系统内存
  3. CPU 调用操作系统的网络协议栈（TCP/IP），把数据打包
  4. 数据经过网卡（NIC）发到网线上
  5. 对面网卡收到数据，CPU 再拆包、拷到内存、再拷到 GPU 显存

每一步都有延迟，而且 CPU 全程参与，占用了本该用来干别的事情的计算资源。

RDMA 的做法完全不同：

  1. GPU 算完数据，网卡直接去 GPU 显存里读
  2. 网卡直接把数据发到对面网卡
  3. 对面网卡直接把数据写到目标 GPU 的显存里

CPU 全程不参与。数据路径极短，延迟极低。

生活类比：
普通网络像寄信——写内容、装信封、贴邮票、投邮筒、邮局分拣、运输、派送、拆信。每一步都有人经手。
RDMA 像直接把东西递到对方手上——没有中间环节，快得多。

InfiniBand 与 RoCE：两条技术路线

RDMA 是一种"能力"，不是一种具体的硬件。实现 RDMA 有两条主流技术路线：

InfiniBand（IB，无限带宽）

专为高性能计算（HPC）设计的网络技术。从硬件到软件完全自成一派，不走以太网协议。需要专用的 InfiniBand 交换机、网卡和线缆。延迟可以做到约 1 微秒（百万分之一秒）。这是 AI 训练集群的"黄金标准"。NVIDIA 旗下的 Mellanox 是这个市场的绝对霸主。

RoCE v2（融合以太网上的 RDMA）

不需要专用硬件——在标准以太网上跑 RDMA。成本低，兼容现有数据中心网络基础设施。延迟约 5-10 微秒，比 InfiniBand 高一些，但对很多场景已经够用。随着以太网速度提升到 400G、800G，RoCE 正在快速追赶 IB。

实际情况：今天绝大多数万卡级的大模型训练集群用的是 InfiniBand。但很多云厂商（如 AWS、阿里云）在推 RoCE 方案，因为以太网交换机便宜、运维人员好找、能和云上的其他业务共用网络基础设施。

三个关键指标

带宽（Bandwidth）

每秒能传多少数据。目前主流是每个端口 400 Gbps（吉比特每秒），正在向 800 Gbps 过渡。8 张 GPU 共享一个 400G 出口和独享 400G 是两种完全不同的体验。

延迟（Latency）

一个消息从发出到收到的耗时。InfiniBand 可到 1 微秒左右。延迟越低，GPU 空闲等数据的时间越短。

尾延迟（Tail Latency）

最慢那 1% 的消息的延迟。All-Reduce 要等所有 GPU 的数据都到位才能继续，所以训练速度被"最慢的那个"拖累。尾延迟比平均延迟更关键。

技术要点

网络拓扑：GPU 怎么连在一起？

8000 张 GPU 不是随便用网线连起来就行。怎么连（网络拓扑）直接决定性能和扩展性。

Leaf-Spine（叶脊架构）

目前最主流的 AI 集群拓扑。结构很简单：

  • Leaf 交换机（叶子层）：直接连接 GPU 服务器。每台服务器有两个网卡，连到两台不同的 Leaf 交换机（冗余）。
  • Spine 交换机（脊层）：每台 Spine 和每台 Leaf 都有一条链路。这样任意两台服务器之间最多经过 3 跳（Server → Leaf → Spine → Leaf → Server），延迟可预测。

好处：扩展方便。要加服务器，就加 Leaf 交换机；带宽不够用了，就加 Spine 交换机。延迟固定，不会随着规模增长而恶化。

DragonFly（蜻蜓拓扑）

更复杂的拓扑结构，用于超大规模集群。核心思路是用更少的线缆覆盖更多节点——把交换机分成"组"，组内全互联，组间部分互联。比 Leaf-Spine 省线缆，但路由更复杂，管理难度更高。

目前绝大多数 AI 训练集群用 Leaf-Spine 就够，DragonFly 主要用在超算中心。

光模块：机房里的"数据高速路"

服务器和交换机之间用铜缆（DAC，直连铜缆）就够了——几米距离，便宜可靠。但交换机之间、机柜之间，距离可能几十到几百米，铜缆信号衰减严重，必须用光纤。而光模块就是负责"电信号和光信号互相转换"的器件。

简单说：光模块插在交换机的端口上，一头接收电信号，把它变成光信号通过光纤发出去；另一头把收到的光信号变回电信号。

400G 向 800G 的过渡

光模块的速度跟着交换机端口走：

  • 2018-2022：100G → 200G → 400G
  • 2023-2024：400G 大规模部署
  • 2024-2025：800G 开始量产，1.6T（1600G）在研发中

速度翻倍意味着什么？同样训练一个模型，如果 All-Reduce 通信时间占训练总时间的 40%，把网络带宽翻倍就能把这 40% 减半，等效于训练速度提升约 20%——几千张 GPU，提 20% 是什么概念。

NVIDIA 的端到端方案：最深的护城河

看懂 NVIDIA 为什么这么强，不仅要看 GPU，更要看它的"全链路控制"：

NVIDIA 数据路径：
GPU ← NVLink（片间互联） → NVSwitch（机内 GPU 全互联） ← ConnectX 网卡（RDMA 智能网卡） → Quantum 交换机（InfiniBand）/ Spectrum 交换机（以太网） → 其他 GPU

从芯片到网线，NVIDIA 控制了数据经过的每一个节点：

  • NVLink + NVSwitch：服务器内部 8 张 GPU 之间的互联。最新一代 NVSwitch 让机内 GPU 之间带宽达到 1.8 TB/s。
  • ConnectX 智能网卡：RDMA 网卡，直接读写 GPU 显存。支持 InfiniBand 和 RoCE 双协议。最新 ConnectX-8 支持 800 Gbps。
  • Quantum 交换机：InfiniBand 交换机，NVIDIA 的看家产品。Quantum-3 支持 400 Gbps 单端口，Quantum-X800 支持 800 Gbps。
  • Spectrum 交换机：以太网交换机产品线，跑 RoCE。面向不想用 InfiniBand 的客户。
关键事件：2019 年 NVIDIA 以 69 亿美元收购 Mellanox。当时很多人觉得贵，今天回头看，这是 NVIDIA 历史上最重要的战略决策之一。它让 NVIDIA 从"只卖 GPU"变成了"卖整个 AI 基础设施"。客户买 GPU 就必须买配套网络，竞争对手在任何一个环节都很难替代。

这种垂直整合是 NVIDIA 最深的结构性优势——即使有人做出比 H100 更强的 GPU，也很难复制从 NVLink 到 Quantum 交换机的全栈深度。

怎么选：给非技术决策者的建议

如果你是销售或采购，面对客户问"网络怎么配"，几个核心判断依据：

  • 规模决定协议：1000 张 GPU 以上的训练集群，InfiniBand 几乎是必选。几百张卡规模，RoCE 完全可以。推理场景（GPU 之间不需要频繁通信）用 RoCE 甚至普通以太网都行。
  • NVIDIA 捆绑策略：买 NVIDIA GPU，网络大概率也是 NVIDIA 的。这也是 NVIDIA 收购 Mellanox 的目的——让客户在一个供应商这里买齐。
  • 国产替代路径：华为昇腾 + CloudEngine 交换机 + 自研光模块，是目前唯一能做到"全国产"的方案。性能有差距，但自主可控是硬需求。
  • 光模块是耗材：光模块有寿命，故障率比交换机高得多。大集群通常需要储备 3%-5% 的备件。

阶段 3 总结

AI 训练集群中，网络不是配角——它直接决定了几千张 GPU 能发挥多少性能。

  • RDMA 让数据绕开 CPU 直达 GPU 显存，是高速网络的基础
  • InfiniBand 是当前万卡训练集群的标配，延迟约 1 微秒
  • RoCE v2 在以太网上跑 RDMA，成本更低，正在快速追赶
  • 网络设备占集群总成本 15%-25%，不是"配件"而是一级决策
  • NVIDIA 通过收购 Mellanox 控制了从 GPU 到交换机的全链路，这是它最强的竞争壁垒之一
